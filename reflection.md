# Reflection

This project made the GIGO principle very visible: the quality of the chatbot depends directly on the quality of the system prompt, not just the model behind it. A generic prompt such as "act like a mentor" produces generic motivation, but a prompt with background, values, tone, constraints, and few-shot examples gives the model a much clearer behavioral target. The best results came from treating each persona prompt like product design: define the audience, decide what the assistant should optimize for, and make the desired response pattern explicit.

What worked best was separating the three personas by decision style rather than only by biography. Anshuman's prompt focuses on first-principles reasoning and Socratic pressure. Abhimanyu's prompt focuses on shipping, users, proof of work, and practical career outcomes. Kshitij's prompt focuses on invariants, edge cases, complexity, and precise DSA thinking. That separation helps the same user question produce meaningfully different answers depending on the selected persona.

The few-shot examples were especially useful because they demonstrate tone and structure better than abstract instructions alone. They show how long the response should be, how direct the mentor should sound, and how each answer should end with a question that moves the learner forward. The constraints also matter because these are real people; the bot should represent them professionally, avoid private claims, and not pretend to be the actual person outside an educational simulation.

If I improved this further, I would add a small research appendix with public source links for each persona, then iterate prompts after testing with people familiar with Scaler's teaching style. I would also add persistent chat history, streaming responses, automated prompt regression tests, and deployment monitoring. The current version is functional, but prompt quality is never finished; it improves when real users ask messy questions and the failures are fed back into better examples and clearer constraints.
